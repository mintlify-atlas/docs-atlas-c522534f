## Welcome to CloudNativePG

**CloudNativePG (CNPG)** is an open-source Kubernetes operator designed to manage PostgreSQL workloads natively in Kubernetes. It covers the entire operational lifecycle — from deploying a cluster to automated failover, backups, upgrades, and monitoring — without relying on external HA tools.

CloudNativePG was originally built by [EDB](https://www.enterprisedb.com) and is now a [CNCF Sandbox project](https://www.cncf.io/sandbox-projects/).

## How it works

CloudNativePG introduces the `Cluster` Custom Resource Definition (CRD), which represents a PostgreSQL cluster in Kubernetes. The operator continuously reconciles the desired state defined in the `Cluster` resource with the actual state of the running PostgreSQL instances.

<Steps>
  <Step title="Install the operator">
    Deploy the CloudNativePG operator into your Kubernetes cluster using a YAML manifest or the `cnpg` kubectl plugin.
  </Step>
  <Step title="Define a Cluster">
    Create a `Cluster` resource specifying the number of instances, storage size, PostgreSQL version, and configuration.
  </Step>
  <Step title="Connect your applications">
    Use the operator-managed services (`-rw`, `-ro`, `-r`) to connect applications to the primary or replicas.
  </Step>
  <Step title="Operate with confidence">
    The operator handles failovers, rolling updates, backups, and scaling automatically according to your desired state.
  </Step>
</Steps>

## Custom Resource Definitions

CloudNativePG manages the following Kubernetes resources:

| CRD | Purpose |
|-----|---------|
| `Cluster` | PostgreSQL primary/replica cluster |
| `Backup` | On-demand backup of a cluster |
| `ScheduledBackup` | Recurring backup schedule |
| `Pooler` | PgBouncer connection pooler |
| `Database` | Declarative PostgreSQL database |
| `Publication` | Logical replication publication |
| `Subscription` | Logical replication subscription |
| `ImageCatalog` | Namespace-scoped image catalog |
| `ClusterImageCatalog` | Cluster-scoped image catalog |
