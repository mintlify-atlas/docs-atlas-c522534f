## Additional CRDs

| CRD | Purpose |
|-----|---------|
| `Database` | Declarative management of PostgreSQL databases, extensions, schemas, and FDWs |
| `Publication` | Logical replication publication |
| `Subscription` | Logical replication subscription |
| `ImageCatalog` | Namespace-scoped catalog of PostgreSQL container images |
| `ClusterImageCatalog` | Cluster-scoped catalog of PostgreSQL container images |
| `FailoverQuorum` | Tracks quorum state for quorum-based failover (managed by operator) |

## API group

All CRDs use the API group `postgresql.cnpg.io/v1`.

Example resource header:

```yaml
apiVersion: postgresql.cnpg.io/v1
kind: Cluster
metadata:
  name: my-cluster
  namespace: default
```
